## Design System Checklist

### Foundation Layer
- [ ] Design tokens defined (colors, spacing, typography, shadows)
- [ ] Token naming convention established
- [ ] Base grid system configured
- [ ] Breakpoint strategy documented
- [ ] Color contrast validated (WCAG AA minimum)

### Component Layer
- [ ] Atomic components created (buttons, inputs, icons)
- [ ] Molecular components composed (cards, forms, nav items)
- [ ] Organism components designed (headers, footers, sections)
- [ ] All states documented (default, hover, focus, active, disabled)
- [ ] Dark mode variants created

### Documentation Layer
- [ ] Component usage guidelines written
- [ ] Do/Don't examples provided
- [ ] Accessibility requirements documented
- [ ] Code snippets included
- [ ] Version changelog maintained

## Accessibility Audit Checklist

### Perceivable
- [ ] All images have meaningful alt text
- [ ] Color is not the only means of conveying information
- [ ] Text has 4.5:1 contrast ratio (AA) or 7:1 (AAA)
- [ ] Content can be resized to 200% without loss
- [ ] Captions available for video content

### Operable
- [ ] All functionality available via keyboard
- [ ] Focus indicators visible and clear
- [ ] No keyboard traps exist
- [ ] Skip links provided
- [ ] Touch targets minimum 44x44px

### Understandable
- [ ] Language of page declared
- [ ] Error messages are clear and helpful
- [ ] Form labels are associated with inputs
- [ ] Navigation is consistent
- [ ] Help is available for complex interactions

### Robust
- [ ] Valid HTML structure
- [ ] ARIA used correctly when needed
- [ ] Works with assistive technologies
- [ ] Tested with screen readers

## Color Palette Creation Guide

### Step 1: Define Brand Colors
```
Primary:   Main brand action color
Secondary: Supporting actions
Accent:    Highlights and emphasis
```

### Step 2: Create Semantic Colors
```
Success:   #10B981 (green family)
Warning:   #F59E0B (amber family)
Error:     #EF4444 (red family)
Info:      #3B82F6 (blue family)
```

### Step 3: Generate Scale (50-950)
```
50:  Lightest (backgrounds)
100: Very light
200: Light
300: Light medium
400: Medium light
500: Base color
600: Medium dark
700: Dark
800: Very dark
900: Darkest (text on light)
950: Near black
```

### Step 4: Validate Contrast
| Usage | Minimum Ratio |
|-------|---------------|
| Body text | 4.5:1 |
| Large text (18px+) | 3:1 |
| UI components | 3:1 |
| Focus indicators | 3:1 |

## User Research Templates

### Interview Script Template
```
1. INTRO (2 min)
   - Introduce yourself and purpose
   - Explain recording/note-taking
   - "There are no wrong answers"

2. WARM-UP (3 min)
   - Background questions
   - Current behavior and context

3. CORE QUESTIONS (20 min)
   - Open-ended questions
   - Follow-up probes
   - Specific scenarios

4. WRAP-UP (5 min)
   - Summary confirmation
   - Additional thoughts
   - Thank participant
```

### Usability Test Plan
```
Objective:  [What you're testing]
Hypothesis: [What you expect to find]
Participants: [5-8 users matching persona]
Tasks:      [3-5 realistic scenarios]
Metrics:    [Completion rate, time, errors, satisfaction]
```
